Explaining the use of Ruby classes

This week we have been focusing on classes, class variables, instance variables and instance methods, I have written a short program to model an airline booking system (in a very simple way!).


My example

The program above will take in the customers surname, number of people travelling and number of children, it is designed to return row numbers in the format of a number between 1 and 100 followed by a seat letter between A and G. Once the seats are assigned it will return the information needed by the customer, such as seat numbers and booking reference related to their surname.

Explanation

The most succinct explanation of classes and instance methods I found is 'Class methods are methods that are called on a class and instance methods are methods that are called on an instance of a class' fromwww.railstips.org. How I understand this is that you define a class and within that class there are class methods and instance methods.

Class

Class methods are used on something that doesn’t deal with an individual instance of a class, so it doesn’t need to be run on every instance of class we might come across.

To denote a class VARIABLE you would use @@ before the variable, '@@variable_name = x' this makes this variable accessible to all methods(class and instance) across the class but each will inherit the value such as '@@variable_name = x'. In my example above @@letters = ['A', 'B', 'C', 'D', 'E', 'F', 'G', 'H'] lets the other methods inherit this variable and its values. If it is changed later on, it will stay equal to that changed value for all methods in class.

Instance

Instance METHODS are used on something that DOES deal with an individual instance of a class. To denote a instance VARIABLE you would use @ before the variable, @variable_name = x, this means this variable is equal to x only for each instance of x and does not permanently change the value of x. An instance variable is also available across all methods in class.

In my example above @surname = surname means that @surname is only equal to surname for each time a surname is passed in, it does not keep the same value. If the input surname in the initialize method is changed, so will the value of @surname. The @ means that other methods can access the value of that variable. So in the Booking.getseats method it accesses @numberchildren which is an instance variable of the initialise method. The value of @numberchildren changes depending on the input value of numberchildren in initialize method.

Further Information

Initialize method is an instance method which defines how you create an object(of said class) in the first place. My initialize method defines a new booking as surname, number of adults travelling and number of children travelling. Initialize is always private so can not be accessed outside of the class but the other methods in class may refer to its variables or methods.

Defining a class does not run any code or use it in anyway, after you have defined the class it needs to be used by creating a new object of the class, in my example I have created three new objects booking_one, booking_two and booking_three which then uses the class Booking to create three new objects based on the information in initialize.

Conclusion

Classes are used when you have many different instances of an object, it could be lots of different people with values of hair colour, weight, height or a booking form like mine, a class groups together methods to be placed on a particular group of objects. Once you have written various different classes you can use then in other files by using 'require file_name' at the top of the file you are wanting to use the classes in. They are useful to use as references later and can be linked together to evaluate lots of different information or lots of the same information differently.
