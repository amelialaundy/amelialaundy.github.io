The Three Musketeers


Athos, Porthos and Aramis, or Blocks, Procs and Lambdas

Blocks, procs and lambdas are all different but similar ways of passing code to a method, and then executing that code. Primarily this is what a block does and you write that block each time you want to call some specific code.

I will endeavour to explain these differences in the following blog post!

Blocks

Blocks are defined by either having do *something* end or { |something| do code to something }. They are chunks of code that have no name and are written when they are called. Often they are used to iterate a variable by using a method such as #each or can be added to a method that is not a built in method by calling it using yield:

Blocks are not objects and so can not be reused and applied over and over, I think of them as pieces of code that can not be saved.

Lambdas

So if procs (are/)contain blocks, what are lambdas all about you ask?

Lambdas are pretty much blocks!

Secondly when lambdas are called within a method, they return their value to the method and then allow the method to continue, methods calling procs on the other hand will, when they meet a return statement in the proc, return the provided value from the proc and won’t continue through the rest of the method.



In Conclusion

  • Procs and lambdas save block of code for reuse
  • Procs and lambdas can both be called using the .call method
  • Blocks can be called using yield
  • Lambdas and blocks have diminutive returns, a method will continue processing after after calling either of them
  • Procs have strong returns, a method calling a proc will stop processing once it hits the return statement in a proc
