Enumerable#group_by


What does group_by do?

group_by groups the collection according to the result of the block, this enables you to create a hash from the block passed after you call group_by on an object. The hash keys are the result of the block and the values are arrays of elements in the collection that correspond to the key.

Examples

Bear with me! This might sound confusing, I find it hard to understand concepts from written wordy explanations, so here is an example:
If you have a list of cars you own and you want to group the list by the first letter of the car you would:

In the block {|x| x[0]} the x evaluates each element in the cars array and x[0] looks at the first letter of each string, thus resulting in x[0] (the first letter of each car) being the key in the hash and the values of each key are any car with that letter as it’s first.

A bit more hardcore

Lets make this a little more complicated. In the following example imagine you are an estate agent and you have asked someone to do some data entry for your listings, you asked them to write an array for the the house sizes (3 bedroom, 4 bedroom etc), but on looking at their work you realise they can't spell very well and have misspelt a lot of the elements of the array. This code will find any (known) spelling mistakes and any correct spellings and result in two hashes, one giving you the elements spelt correctly and another with the elements with misspellings. You can now take the misspelling hash and use this to correct the mistakes.

Brief explanation:
In the case statement we are looking at each spelling(string in the array) and saying "when it contains Bedrom or bedromm or bedrom create a key "Typo" that takes the corresponding strings as it’s values", when it contains bedroom "create a key "Correct" that takes these corresponding strings as it’s values". The else "Correct" in this example will return any string from the house_sizes array that contains anything else not mentioned, this is why 6+ Bedroom is a value in the "Correct" key as the capital b wasn’t mentioned in the when condition "Correct" only a lower case b was.
