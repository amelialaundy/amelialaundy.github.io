Why does Javascript hold the Miss Popularity prize?

Javascript as the future

"As long as there are people using the internet, javascript will be alive" or something similar was a quote I came across the other day and this sparked me to really question the validity behind this prediction/claim because due to being a Javascript 'speaker' for approximately 6 whole days now I have no knowledge of the popularity or longevity of this language.

Everyone wants to be friends with JS

I found this article on tech crunch whilst researching why javascript is so popular and was found the results of their ranking system to be quite succinct.

Now you do need to take into consideration that it is difficult to determine qualitatively the popularity of a programming language, but these guys(RedMonk) have analysed the programming languages used in files on GitHub and data from the Q & A side StackOverflow so it seems to me a fair way to judge popularity. The results speak for themselves that Javascript is top of the list followed by Java, PHP, Python and Ruby (waheyy!) as the top 5. Ok, Javascript it seems is popular, but why?

Possibly because of these reasons!

  • It's easy to learn!
    Javascript is a flexible language, and comes standardized on most common browsers meaning that it is also far more useful.
  • It is easy to edit (and cheap!).
    Javascript is a scripting program so you dont need to compile it or buy expensive compiling software, you only need a text editor such as sublime text 2 that I'm using here.
  • There are lots of tools available.
    Browsers now have built in options giving pwer to the user, you can view and edit the javascript of your favourite webpage right there. For example the Web Developer Toolbar on chrome allows the user to test javascript without having to worry about savings, uploading anything.
  • Debugging is easy.
    With the tools available it is easy to debug javascript, you can trace back to the exact place the error occurred in the browser console.
  • Javascript is easily extensible.
    You cando complex things behind the scenes or manipulate the aesthetics with Javascript, but this can be very time consming and takes a lot of code to develop. So developers have written several libraries to support Javascript such as jQuery, Prototype and Dojo that allow and support Javascript to manipulate pages with only a few lines of code.
  • AJAX makes Javascript even better!
    AJAX (asynchronous Javascript and XML) allows browsers to request information from the server without leaving or reloading the page. Before this addition anything that needed to change on a webpage would have to be requested via reloading the page.

One quote I feel sums up the pro's of Javascript: "In short, JavaScript is quickly becoming not just the ideal language for the web, but indeed a big composition of the web itself. The future of the web looks bright, and perhaps more importantly free for developers everywhere."

Only human:the limitations

The saying we're only human refers to the impossibilty of being 100% perfect, neither is javascript and like the rest of the popular kids it comes with some flaws

There are two main issues with Javascript that are somewhat limiting:

  • There are security issues.
    Malicious code can be executed due to the fact that Javascript code is run on the users computer.
  • The rendering can vary.
    Different browsers may render Javascript differently meaning an inconsistency in the interface and functionality. Ideally there needs to be a universal standard and these are being worked towards but variations still do exist.
